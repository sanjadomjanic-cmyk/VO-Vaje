# Policy Pack (Minimum Viable Set)

## 1. Access Control Policy

- All users must have unique accounts (no shared accounts)
- Access is granted based on least privilege principle
- Multi-factor authentication (MFA) is required for VPN and cloud services
- User access must be reviewed quarterly
- Accounts must be disabled immediately upon employee termination

## 2. Backup & Recovery Policy

- All critical systems must be backed up daily
- Backups must be stored in a secure and separate location 
- Backup restoration must be tested quarterly
- Backup data must be protected from unauthorized access and ransomware
- Recovery procedures must be documented and regularly updated

## 3. Incident Response Policy

- All security incidents must be reported immediately to IT
- Incidents must be logged, classified, and prioritized
- A response team must be assigned roles and responsibilities
- Systems affected by incidents must be isolated promptly
- Data breaches must be reported in accordance with GDPR (within 72 hours

## 4. Vendor Security Policy

- All third-party vendors must undergo a security assessment before onboarding
- Vendors must comply with GDPR and healthcare regulations
- Access provided to vendors must follow least privilege principle
- Contracts must include data protection and security requirements
- Vendor performance and security must be reviewed periodically

## 5. Acceptable Use Policy

- IT systems must be used for business purposes only
- Users must not install unauthorized software
- Users must not share passwords or accounts
- Suspicious emails or activities must be reported immediately
- Devices must be locked when unattended
